# 1 项目整体概述

## 1.1 项目背景与核心定位

本项目为**社区活动与商家赞助智能供需对接平台**，聚焦解决社区日常活动开展痛点与本地商家精准获客难题，搭建正规、合规、闭环的第三方撮合渠道。一方面，居委会、物业、社区社团等主体举办节日庆典、亲子互动、健康科普、文化娱乐、志愿服务等活动时，普遍面临经费不足、物资短缺、专业人力匮乏、场地资源有限等问题，急需稳定的外部资源支持；另一方面，本地餐饮、母婴、教培、健康、商超、银行等商户，拥有精准触达社区居民的获客需求，但缺乏合规、高效、可落地的社区准入渠道，盲目进场易引发社区居民反感，也无法实现精准转化。

平台核心定位为轻量化、闭环式资源对接工具，不涉及复杂社区生活服务，仅聚焦活动需求与赞助资源撮合，通过标签化智能匹配替代人工对接，同时搭建完整的线上合作闭环、防飞单机制、商家评价体系，保障平台、社区、商家三方权益，实现社区活动顺利落地、商家精准获客变现、平台稳定盈利的三方共赢目标。

## 1.2 适用终端

- **社区端**：手机网页版（H5），适配安卓、iOS全机型手机浏览器及微信内置浏览器打开，自适应手机屏幕尺寸，操作极简易懂，充分适配中老年社区工作人员日常操作习惯，无需下载安装，点开链接即可使用

- **商家端**：手机网页版（H5），适配安卓、iOS全机型手机浏览器及微信内置浏览器打开，界面简洁、操作步骤少，满足本地商户经营者、业务员快速操作、随时接单报名的需求，无额外安装门槛

- **管理后台**：PC端网页版，适配Chrome、Edge、360浏览器、QQ浏览器等主流桌面浏览器，无需下载客户端，管理员通过账号密码登录即可远程操作，支持多窗口切换、数据批量处理

## 1.3 核心盈利模式

- 商家会员年费：分四档会员，收取会员费用，享受不同匹配权益

- 定制活动服务费：商家在线下单定制社区专场活动，平台收取策划执行费用

- 广告位服务费：优质商家付费获取平台首页、活动列表页推荐位

- 线索与成交提成：精准客户线索售卖、商家成交后抽取小额佣金（可选）

## 1.4 核心约束要求

平台所有业务流程必须实现**全链路线上闭环**，社区与商家的沟通、对接、合作确认、评价等所有行为，均需在平台内完成，严禁任何形式的私下对接、飞单、跳单行为；平台不收集、不泄露社区居民个人隐私信息，仅做社区主体与商家主体的资源对接，不参与活动现场具体执行，全程合规运营。

# 2 用户角色与权限明细

## 2.1 社区用户

**身份定义**：社区居委会、小区物业、社会组织等正规社区主体

**核心权限**：微信快捷登录与手机号验证；完成实名认证并提交资质证明、社区基本情况档案；发布社区活动需求；查看系统智能匹配的商家列表；查看商家赞助报名信息；线上确认或拒绝合作；查看合作进度与历史记录；活动结束后上传现场照片、填写小结；对合作商家进行星级评分与评价；接收平台系统通知与对接消息；无付费权限，不可购买会员与服务；支持在个人中心修改更新社区基本情况信息，修改记录留存后台备查。

**权限限制**：未实名认证、未完整填写社区基本情况的用户无法发布活动；无法查看未合作商家的完整联系方式；不可主动联系未报名商家；不可修改平台活动与商家信息；不可查看、修改平台补充的活动图文内容；社区基本情况信息需如实填写，不得虚报瞒报，违规将限制活动发布权限；触发防飞单规则将按等级处罚。

## 2.2 商家用户

**身份定义**：本地正规注册商户、企业、门店，具备营业执照与合法经营资质

**核心权限**：微信快捷登录与手机号验证；完成企业认证并上传营业执照；完善商家资料与匹配标签；查看系统智能推荐的高匹配活动；浏览活动广场所有活动；报名活动赞助；购买平台会员套餐；下单定制社区活动服务；查看合作进度与历史订单；接收平台智能推送与消息通知；查看社区确认后的合作联系方式；接受社区用户的星级评价；查看自身星级与评价内容；查看违规处罚记录。

**权限限制**：未认证用户无法报名活动；免费会员每日报名活动有次数限制；无法查看未合作社区的完整联系方式；不可发布活动需求；不可修改平台活动信息；违规用户会被限制权限直至封号；严禁私下对接、引导飞单，违规直接处罚。

## 2.3 平台管理员

**身份定义**：平台运营方工作人员，分超级管理员与普通运营管理员

**核心权限**：后台账号登录；审核社区用户资质、商家资质、活动内容、社区基本情况信息；管理所有用户账号与活动信息；手动干预智能匹配结果，定向推荐商家与活动；**对已审核通过的社区活动，补充编辑图文介绍（含文字描述、活动宣传图、活动海报等），优化活动展示效果，编辑记录留存后台**；监控用户沟通内容，排查飞单、违规导流行为；处理用户投诉与纠纷；对违规用户进行分级处罚并记录；管理会员套餐、定制活动价格；查看财务订单与数据统计；导出数据报表；发布平台公告；配置系统基础参数；查看用户资料修改日志、活动图文编辑日志。

# 4 各模块细分流程图（可视化版）

本章节所有细分流程均采用独立可视化流程图呈现，对应各功能模块的独立业务逻辑，精准对应前端操作页面与后端接口逻辑，每幅流程图标注模块用途，方便程序员分模块开发、单独调试，避免流程混淆，全程剔除重复代码流程图，统一为可视化描述格式。

## 4.1 社区端模块细分流程

- **社区注册认证流程**：登录授权 → 填写社区资质信息 → 填写社区基本情况全量字段 → 上传证明材料 → 提交审核 → 管理员同步审核资质与档案信息 → 审核通过启用功能/驳回修改重提

- **活动发布与归档流程**：填写活动基础信息 → 选择需求与回报 → 提交审核 → 管理员审核通过 → 平台补充活动图文 → 活动上线招募 → 等待商家报名 → 确认合作 → 活动执行 → 上传现场照片 → 填写活动小结 → 完成归档评价

- **社区档案修改流程**：进入个人中心 → 修改社区基本情况 → 提交更新申请 → 管理员审核更新 → 生成修改日志

## 4.2 商家端模块细分流程

- **商家认证与资料完善流程**：登录授权 → 填写商家经营信息 → 上传营业执照 → 选择行业与人群标签 → 提交审核 → 审核通过启用报名功能

- **商家报名与合作流程**：浏览匹配活动 → 查看含平台补充图文的完整活动详情 → 提交赞助报名 → 等待社区确认 → 合作通过解锁联系方式 → 对接执行 → 接收评价

## 4.3 智能匹配模块流程

活动/商家标签录入+社区基本情况标签纳入匹配维度 → 系统按算法计算匹配分数 → 划分匹配等级 → 按规则排序展示 → 管理员可手动干预排名与匹配权重

## 4.4 防飞单模块流程

联系方式默认屏蔽 → 聊天敏感词实时拦截并记录 → 违规行为后台监控标记 → 疑似飞单人工核查 → 按等级分级处罚并生成违规记录

## 4.5 商家星级评价流程

活动归档完成 → 社区进入评价页 → 多维度星级评分 → 填写评价内容 → 提交评价 → 系统自动计算综合星级 → 星级同步展示并影响匹配权重

## 4.6 平台活动图文补充流程

社区活动审核通过 → 管理员进入活动编辑页 → 补充活动文案/上传宣传图片 → 提交发布 → 同步更新至前后端活动详情页 → 生成图文编辑日志

# 5 三端全页面详细内容说明（社区端H5+商家端H5+管理后台）

## 5.1 社区端手机网页版（H5）全页面详情

### 5.1.1 登录注册页

顶部展示平台全称**“邻盟”社区活动与商业资源智能对接助手**，下方展示平台核心slogan“邻盟聚力，资源互通，让社区活动落地更轻松”；中间区域设置角色选择按钮，清晰分为“社区用户入口”“商家用户入口”，按钮采用大尺寸圆角设计，适配手机点击，点击社区用户后，跳转微信快捷授权/手机号登录页面；授权验证通过后，跳转社区实名认证页面，分为基础资质信息与社区基本情况两大板块，需完整填写后方可提交审核。
**基础资质信息**：社区全称、负责人姓名、所属街道、详细地址，上传资质证明照片；
**社区基本情况（必填，用于精准商家匹配）**：社区具体位置（可联动地图定位填写）、居民总户数、老年人占比（填写百分比数值）、青少年家庭占比（填写百分比数值）、社区公共空间总面积（单位：㎡）、所辖小区名称（多个小区用逗号分隔）、辖区商户数量，所有数据需如实填写，提交后平台审核时同步核验，提交后页面实时显示“审核中，请等待管理员短信及站内通知”，审核通过后社区基本情况自动存入档案，支持后续在个人中心修改更新，修改需重新提交审核。

### 5.1.2 社区首页（H5）

顶部导航栏：左侧无返回按钮，中间居中显示页面标题“首页”，右侧放置消息通知铃铛图标，未读消息标红角标；下方平台横幅区：采用邻盟品牌主色调（浅蓝色）背景，展示核心价值文案“社区活动缺资源？邻盟免费对接商家赞助、物资、人力、场地”；快捷操作区：两大核心功能按钮并排展示，左侧“发布活动需求”，右侧“我的活动”，按钮尺寸适配手机单手操作，圆角12px，品牌主色调填充；数据统计区：卡片式布局，展示已发布活动总数、招募中活动数、已完成合作数，数据实时同步后台；智能推荐区：标题“邻盟为你优选合适商家”，横向滚动卡片样式，每张卡片展示商家名称、主营行业、可提供赞助类型、匹配度百分比、一键对接按钮；我的活动区：标题“我的近期活动”，列表展示最近3条活动信息，显示活动标题、举办时间、当前状态，点击“查看更多”跳转完整活动列表页；底部固定导航栏：首页、我的中心、消息中心，图标+文字组合，适配手机底部点击习惯。

### 5.1.3 活动发布页（H5）

# 6 商家星级评价体系（新增核心模块）

## 6.1 星级评分规则

商家星级采用**5星制**，由社区用户在合作完成、活动归档后进行客观评分，评分维度分为三项，每项对应权重，综合计算得出商家最终星级：

- **赞助质量（权重40%）**：赞助物资/现金/人力是否符合承诺，数量、质量是否达标

- **配合度（权重30%）**：活动对接过程中是否及时沟通、配合社区安排、履约到位

- **服务态度（权重30%）**：商家工作人员服务态度、沟通语气是否良好

综合得分计算方式：（赞助质量得分+配合度得分+服务态度得分）÷ 3 ÷ 20，保留1位小数，对应星级：

- 4.5-5.0分：★★★★★（五星优质商家）

- 4.0-4.4分：★★★★☆（四星良好商家）

- 3.0-3.9分：★★★☆☆（三星一般商家）

- 2.0-2.9分：★★☆☆☆（二星较差商家）

- 1.0-1.9分：★☆☆☆☆（一星劣质商家）

## 6.2 星级展示与应用

- 商家星级展示在商家详情页、活动报名列表、智能推荐列表，星级越高展示越靠前

- 星级直接影响智能匹配权重，五星商家匹配权重额外+1，四星商家无加减，三星及以下商家匹配权重-1，二星及以下限制报名活动数量

- 五星商家可获得平台首页推荐资格，优先推送优质社区活动

- 一星商家平台将进行预警，多次差评直接限制报名甚至封号

## 6.3 评价管理规则

- 评价仅社区合作完成后可发布，每个合作单仅可评价一次，不可修改，不可重复评价

- 评价内容需客观真实，严禁恶意差评、辱骂、虚假评价，管理员可删除违规评价，删除后不影响商家星级

- 商家可查看所有评价，但不可修改、删除，可针对恶意评价发起申诉，管理员审核处理

- 评价实时同步，星级每新增一条评价自动重新计算，实时更新

# 7 智能供需匹配引擎完整规则

## 7.1 匹配标签体系

社区活动标签（必选）：面向人群（亲子/少儿、中老年、青年、宝妈、全社区）、需求赞助类型（现金、物资、物料、人力、场地）、活动类型（节日活动、市集、健康科普、亲子互动、文化体育、娱乐休闲、公益志愿、环保、就业招聘、教育培训、外出团建、其他）、场地（室内、室外；场地可容纳人数）

商家标签（必选）：行业标签（教育培训、医院诊所、药店、餐饮小吃、生鲜水果、美业、保健养生、体育健身、银行保险、电信服务、商超零售、快递物流、五金建材、家居装修、家纺布艺、电子电器、家政服务、房产中介、汽车服务、旅游服务、鲜花礼品、电影演出、娱乐休闲、服装服饰、酒店宾馆、茶艺咖啡、宠物服务、眼镜、其他）、目标人群（与活动人群一致）、可提供赞助（与活动需求一致）

## 7.2 匹配打分逻辑（固定算法，无歧义）

匹配总分100分，按权重分配，系统自动计算，不可人工修改（管理员可干预排名）：

- **人群匹配（权重60%）**：活动人群与商家目标人群完全一致得60分，部分匹配得30分，不匹配得0分

- **赞助需求匹配（权重30%）**：每匹配一项需求得10分，最高30分，不匹配得0分

- **行业与活动适配+社区档案匹配（权重10%）**：行业与活动高度相关+社区人群结构适配得10分，一般相关得5分，不相关得0分

## 7.3 匹配等级与排序

- 高匹配：总分≥80分，标红色火焰标签，置顶展示

- 中匹配：总分60-79分，标黄色标签，正常展示

- 一般匹配：总分＜60分，标灰色标签，靠后展示

同等级排序规则：高匹配＞中匹配＞一般匹配；同匹配度下，会员等级高＞会员等级低；同会员等级下，商家星级高＞星级低；同星级下，最新报名＞历史报名。

# 9 数据库核心字段设计

本章节包含平台全核心数据表，**每个字段均标注字段名、数据类型、字段说明、业务用途、约束规则**，完整覆盖社区档案关联展示、会员体系、活动场地标签、商家经营地址、新增商家回报项、平台活动图文补充、需求输入框数据存储、日志溯源、违规记录等全业务需求，无字段遗漏、无重复嵌套、无逻辑错乱，字段命名规范统一，便于前后端开发对接与数据库建表；已完成全表核查，前端新增具体需求输入框功能，与活动表need_detail长文本字段完全适配，所有迭代新增需求均已对应入库，无缺失项、无冲突项。

## 9.1 社区用户表（community_user）

表用途：存储社区用户（居委会、物业等）基础资质、社区基本情况档案、账号状态等全量信息，是社区用户核心数据表，关联活动表、评价表、日志表，支持活动详情页点击社区名称调取完整档案展示。

- **id**：整型（int），主键，自增，唯一标识每条社区用户记录，无业务含义，仅用于数据库关联查询

- **openid**：字符串（varchar），微信授权唯一标识，用于微信快捷登录，非空，不可重复

- **phone**：字符串（varchar），社区负责人手机号，用于登录验证、短信通知，非空，唯一

- **community_name**：字符串（varchar），社区全称，如XX街道XX社区居委会，非空，用于平台展示与资质核验，活动详情页可点击字段

- **person**：字符串（varchar），社区负责人姓名，非空，用于对接沟通与资质审核

- **address**：字符串（varchar），社区办公详细地址，非空，基础资质信息

- **certificate_img**：字符串（varchar），社区资质证明照片存储路径，非空，用于审核核验

- **community_location**：字符串（varchar），社区具体地理位置，支持地图定位回填，可标注经纬度或详细地段，用于商家地域匹配，非空

- **household_total**：整型（int），居民总户数，纯数字，无单位，非空，用于分析社区规模，辅助精准匹配

- **elderly_ratio**：浮点型（float），老年人占比，存储百分比数值（如25.5代表25.5%），非空，用于适配养老、健康类商家匹配

- **teenager_family_ratio**：浮点型（float），青少年家庭占比，存储百分比数值，非空，用于适配亲子、教培类商家匹配

- **public_space_area**：整型（int），社区公共空间总面积，单位㎡，纯数字，非空，用于判断活动场地承载能力

- **jurisdiction_communities**：字符串（varchar），所辖小区名称，多个小区用英文逗号分隔，非空，明确社区管辖范围

- **jurisdiction_merchant_count**：整型（int），辖区内商户总数量，纯数字，非空，用于分析辖区商业生态

- **status**：整型（int），账号状态，0=待审核，1=审核通过，2=审核驳回，3=封禁，默认0，控制用户功能权限

- **create_time**：时间类型（datetime），用户注册提交时间，自动生成，不可修改

- **update_time**：时间类型（datetime），用户信息最后修改时间，自动更新，用于追踪信息变更

## 9.2 活动表（activity）
