# 邻盟智能匹配算法（v2.0）

## 一、算法概述

邻盟智能匹配算法是一个多维度、动态权重的匹配系统，用于计算商家与社区需求之间的匹配度。算法综合考虑商家信息、需求信息、社区特征等多个维度，通过加权计算得出匹配分数，并以红心形式直观展示。

## 二、核心计算逻辑

### 1. 动态权重计算机制

算法采用**动态权重**机制，只计算有数据的维度，确保匹配结果的准确性和公平性。

```
总分数 = Σ(维度分数 × 维度权重) / 总权重
```

### 2. 匹配度分数与红心对应关系

| 匹配分数 | 红心数量 | 匹配等级 |
|---------|---------|---------|
| 80-100分 | ❤️❤️❤️❤️❤️ | 高度匹配 |
| 60-79分 | ❤️❤️❤️❤️ | 较高匹配 |
| 40-59分 | ❤️❤️❤️ | 中等匹配 |
| 20-39分 | ❤️❤️ | 较低匹配 |
| 5-19分 | ❤️ | 低匹配 |
| 0-4分 | 无红心 | 不匹配 |

## 三、匹配维度详解

### 1. 赞助类型匹配（权重：30%）

**计算逻辑**：
- 提取商家可提供的赞助类型
- 提取需求所需的赞助类型
- 计算匹配比例

**示例**：
```
商家赞助类型：物资, 场地, 人力
需求赞助类型：物资, 场地
匹配结果：2/2 = 100分
```

**评分规则**：
- 完全匹配：100分
- 部分匹配：按匹配比例计算
- 无数据：30分（默认值）

### 2. 目标人群匹配（权重：25%）

**计算逻辑**：
- 提取商家服务的目标人群
- 提取需求的目标人群
- 计算匹配比例

**特殊规则**：
- 商家服务"全社区居民"：直接得80分

**示例**：
```
商家目标人群：中老年, 亲子/少儿
需求目标人群：中老年
匹配结果：1/1 = 100分
```

**评分规则**：
- 完全匹配：100分
- 部分匹配：按匹配比例计算
- 无数据：30分（默认值）

### 3. 行业匹配（权重：15%）

**计算逻辑**：
- 提取商家所属行业
- 根据行业映射关系匹配需求类型

**行业映射关系**：
```javascript
{
  '餐饮美食': ['物资', '场地'],
  '教育培训': ['人力', '专业服务', '场地'],
  '医疗健康': ['人力', '专业服务', '物资'],
  '文化娱乐': ['场地', '物资', '人力'],
  '运动健身': ['场地', '物资', '人力'],
  '生活服务': ['人力', '物资', '专业服务'],
  '零售购物': ['物资', '现金'],
  '美容美发': ['人力', '物资', '专业服务']
}
```

**评分规则**：
- 行业直接匹配需求：100分
- 行业相关类型匹配：60分
- 无数据：30分（默认值）

### 4. 位置距离匹配（权重：10%）

**计算逻辑**：
- 使用Haversine公式计算商家与社区的实际距离
- 根据距离远近评分

**距离评分标准**：

| 距离范围 | 分数 |
|---------|-----|
| ≤1公里 | 100分 |
| 1-3公里 | 80分 |
| 3-5公里 | 60分 |
| 5-10公里 | 40分 |
| >10公里 | 20分 |

**计算公式**（Haversine公式）：
```javascript
R = 6371 // 地球半径（公里）
dLat = (lat2 - lat1) × π / 180
dLon = (lon2 - lon1) × π / 180
a = sin²(dLat/2) + cos(lat1×π/180) × cos(lat2×π/180) × sin²(dLon/2)
c = 2 × atan2(√a, √(1-a))
distance = R × c
```

### 5. 社区特征匹配（权重：10%）

**计算逻辑**：
- 分析社区人口结构特征
- 匹配商家目标人群

**评分规则**：
- 基础分：50分
- 社区老年比例>30% + 商家服务中老年：+20分
- 社区亲子家庭比例>40% + 商家服务亲子/少儿：+20分
- 社区总户数>2000：+10分
- 最高分：100分

### 6. 商家信誉评分（权重：5%）

**计算逻辑**：
- 基础分：50分
- 根据商家星级评分加分

**评分规则**：
```
分数 = 50 + (星级评分 / 5) × 30
```

**示例**：
- 5星商家：50 + 30 = 80分
- 4星商家：50 + 24 = 74分
- 3星商家：50 + 18 = 68分

### 7. 会员等级评分（权重：5%）

**评分规则**：

| 会员等级 | 分数 |
|---------|-----|
| 钻石会员（3级） | 100分 |
| 金卡会员（2级） | 80分 |
| 银卡会员（1级） | 60分 |
| 免费会员（0级） | 40分 |

### 8. 标签匹配（权重：5%）

**计算逻辑**：
- 提取商家标签（系统标签 + 自定义标签）
- 提取需求标签（系统标签 + 自定义标签）
- 计算匹配标签数量

**评分规则**：
```
分数 = 匹配标签数 × 10（最高100分）
```

**示例**：
```
商家标签：['亲子活动', '健康讲座', '公益服务']
需求标签：['亲子活动', '健康讲座']
匹配数量：2
分数：2 × 10 = 20分
```

## 四、完整计算示例

### 示例1：高度匹配（5颗心）

**商家信息**：
- 赞助类型：物资, 场地
- 目标人群：中老年, 亲子/少儿
- 行业：餐饮美食
- 位置：距离社区2公里
- 会员等级：金卡会员（2级）
- 星级评分：4.5星

**需求信息**：
- 赞助类型：物资
- 目标人群：中老年
- 社区特征：老年比例35%，总户数2500户

**计算过程**：
```
赞助类型匹配：(1/1) × 100 = 100分 × 30% = 30分
目标人群匹配：(1/1) × 100 = 100分 × 25% = 25分
行业匹配：60分 × 15% = 9分
位置距离：80分 × 10% = 8分
社区特征：80分 × 10% = 8分
商家信誉：77分 × 5% = 3.85分
会员等级：80分 × 5% = 4分

总权重：30 + 25 + 15 + 10 + 10 + 5 + 5 = 100
总分数：(30 + 25 + 9 + 8 + 8 + 3.85 + 4) = 87.85分
最终分数：88分
红心数量：5颗心 ❤️❤️❤️❤️❤️
```

### 示例2：中等匹配（3颗心）

**商家信息**：
- 赞助类型：人力
- 目标人群：全社区居民
- 行业：教育培训
- 位置：距离社区6公里
- 会员等级：银卡会员（1级）
- 星级评分：3星

**需求信息**：
- 赞助类型：物资, 场地
- 目标人群：中老年

**计算过程**：
```
赞助类型匹配：(0/2) × 100 = 0分 × 30% = 0分
目标人群匹配：80分（全社区居民） × 25% = 20分
行业匹配：60分 × 15% = 9分
位置距离：40分 × 10% = 4分
商家信誉：68分 × 5% = 3.4分
会员等级：60分 × 5% = 3分

总权重：30 + 25 + 15 + 10 + 5 + 5 = 90
总分数：(0 + 20 + 9 + 4 + 3.4 + 3) = 39.4分
最终分数：39.4 / 90 × 100 = 43.78分
红心数量：3颗心 ❤️❤️❤️
```

## 五、算法优势

### 1. 动态权重机制
- 只计算有数据的维度，避免数据缺失导致的偏差
- 权重根据实际数据动态调整，更加公平合理

### 2. 多维度综合评估
- 8个维度全方位评估匹配度
- 核心维度（赞助类型、目标人群）权重更高
- 辅助维度（位置、社区特征）提供补充信息

### 3. 灵活的评分规则
- 支持完全匹配、部分匹配、无数据等多种情况
- 默认值机制确保无数据时仍能给出合理分数

### 4. 直观的可视化展示
- 红心数量直观展示匹配等级
- 用户无需理解复杂分数即可判断匹配程度
