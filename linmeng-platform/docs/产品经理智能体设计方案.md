# 产品经理智能体设计方案

## 一、产品经理智能体职责

产品经理智能体是整个开发项目的核心协调者，负责：
1. 项目整体规划和需求梳理
2. 智能体分工协调
3. 文档管理和版本控制
4. 质量把控和进度跟踪
5. 用户反馈收集和产品迭代

## 二、智能体团队分工

### 2.1 产品经理智能体 (PM-Agent)
**职责：**
- 需求分析和产品规划
- 功能设计和原型评审
- 文档编写和维护
- 跨智能体协调沟通

**输出物：**
- 需求说明书
- 产品使用说明书
- FAQ文档
- 工作手册

### 2.2 架构师智能体 (Arch-Agent)
**职责：**
- 系统架构设计
- 技术选型决策
- 数据库设计
- API接口设计

**输出物：**
- 系统架构文档
- 数据库设计文档
- API接口文档
- 开发说明书

### 2.3 前端开发智能体 (FE-Agent)
**职责：**
- 前端页面开发
- UI组件实现
- 前端性能优化
- 移动端适配

**负责模块：**
- portal（统一入口）
- community-h5（社区端）
- merchant-h5（商家端）
- salesman-h5（招商大使端）
- admin-web（管理后台）

### 2.4 后端开发智能体 (BE-Agent)
**职责：**
- 后端服务开发
- 数据库操作
- 业务逻辑实现
- 接口开发和维护

**负责模块：**
- 用户认证模块
- 活动管理模块
- 赞助管理模块
- 消息系统模块
- 提成系统模块

### 2.5 测试智能体 (QA-Agent)
**职责：**
- 测试用例编写
- 功能测试执行
- Bug报告和跟踪
- 回归测试

**输出物：**
- 测试用例文档
- 测试报告
- Bug列表

### 2.6 UI设计智能体 (UI-Agent)
**职责：**
- UI界面设计
- 交互原型设计
- 视觉规范制定
- 设计资源输出

**输出物：**
- UI设计稿
- 交互原型
- 设计规范文档
- 图标/图片资源

### 2.7 运维智能体 (Ops-Agent)
**职责：**
- 部署环境配置
- 服务监控告警
- 日志分析
- 备份恢复

## 四、工作流程

### 4.1 需求开发流程
1. **需求收集** → 产品经理智能体
2. **需求分析** → 产品经理智能体 + 架构师智能体
3. **技术评审** → 架构师智能体
4. **开发实现** → 前端智能体 + 后端智能体
5. **测试验证** → 测试智能体
6. **部署上线** → 运维智能体
7. **文档更新** → 产品经理智能体

### 4.2 Bug修复流程
1. **Bug报告** → 测试智能体
2. **问题定位** → 后端/前端智能体
3. **修复实现** → 对应开发智能体
4. **回归测试** → 测试智能体
5. **文档更新** → 产品经理智能体

## 五、沟通机制

### 5.1 日报制度
- 各智能体每日汇报工作进度
- 产品经理智能体汇总并更新项目状态

### 5.2 周会制度
- 每周进行项目进度评审
- 讨论遇到的问题和解决方案

### 5.3 文档管理
- 所有文档统一存放在 `/docs` 目录
- 文档命名规范：`类型_名称_日期.md`
- 版本控制通过Git管理

## 六、质量标准

### 6.1 代码质量
- 遵循代码规范
- 必要的注释
- 单元测试覆盖

### 6.2 文档质量
- 内容完整准确
- 格式规范统一
- 及时更新维护

### 6.3 产品质量
- 功能符合需求
- 性能满足要求
- 用户体验良好

## 七、项目信息

### 7.1 项目名称
邻盟 - 社区爱心资源智能匹配助手

### 7.2 项目目标
打造一个连接社区与商家的智能资源匹配平台，帮助社区活动获得商家赞助支持。

### 7.3 核心价值
- **社区端**：免费获取商家赞助、物资、人力、场地等资源
- **商家端**：精准触达社区居民，提升品牌形象，高效获客转化
- **招商大使**：推广获客，赚取提成
- **平台方**：构建社区商业生态，实现多方共赢
