## 一、项目概述

### 1.1 项目简介
邻盟是一个连接社区与商家的智能资源匹配平台，帮助社区活动获得商家赞助支持，同时帮助商家精准触达社区居民。

### 1.2 项目目标
- 打造社区与商家的资源对接桥梁
- 实现智能匹配，提高对接效率
- 构建多方共赢的商业生态

### 1.3 核心功能
- 社区活动发布与管理
- 商家赞助发布与管理
- 智能匹配推荐
- 合作对接管理
- 招商大使推广系统

## 三、开发规范

### 3.1 代码规范

#### 3.1.1 命名规范
| 类型 | 规范 | 示例 |
|-----|------|------|
| 文件名 | 小写连字符 | `user-profile.vue` |
| 组件名 | 大驼峰 | `UserProfile` |
| 变量名 | 小驼峰 | `userName` |
| 常量名 | 大写下划线 | `API_BASE_URL` |
| 函数名 | 小驼峰 | `getUserInfo` |

#### 3.1.2 注释规范
```javascript
/**
 * 函数说明
 * @param {Type} name 参数说明
 * @returns {Type} 返回值说明
 */
```

#### 3.1.3 Git提交规范
```
feat: 新功能
fix: 修复bug
docs: 文档更新
style: 代码格式
refactor: 代码重构
test: 测试相关
chore: 构建/工具
```

### 3.2 分支管理
```
main        # 主分支，生产环境
develop     # 开发分支
feature/*   # 功能分支
hotfix/*    # 紧急修复分支
```

### 3.3 代码审查
- 所有代码合并前需经过审查
- 审查要点：代码规范、逻辑正确、性能优化
- 审查通过后方可合并

## 四、开发流程

### 4.1 需求开发流程

#### 步骤1：需求分析
- 产品经理智能体收集需求
- 编写需求说明书
- 组织需求评审

#### 步骤2：技术设计
- 架构师智能体进行技术设计
- 编写技术方案
- 设计数据库和API

#### 步骤3：开发实现
- 前端智能体开发页面
- 后端智能体开发接口
- 联调测试

#### 步骤4：测试验证
- 测试智能体编写测试用例
- 执行功能测试
- 提交Bug报告

#### 步骤5：部署上线
- 运维智能体配置环境
- 部署服务
- 监控验证

### 4.2 Bug修复流程

```
发现Bug → 记录Bug → 分配处理 → 修复代码 → 测试验证 → 关闭Bug
```

### 4.3 紧急发布流程
1. 从main分支创建hotfix分支
2. 修复问题并测试
3. 合并到main和develop
4. 部署上线

### 8.1 监控指标
| 指标 | 阈值 | 告警级别 |

| CPU使用率 | >80% | 警告 |
