## 七、智能匹配算法

### 7.1 算法概述
系统采用多维度匹配算法，根据活动需求和赞助信息进行智能匹配。

### 7.2 匹配维度
| 维度 | 权重 | 说明 |
|-----|------|------|
| 类型匹配 | 30% | 活动类型与赞助类型匹配度 |
| 资源匹配 | 25% | 所需资源与提供资源匹配度 |
| 地理位置匹配 | 20% | 距离远近 |
| 时间匹配 | 15% | 活动时间与赞助有效期 |
| 历史合作 | 10% | 历史合作记录 |

### 7.3 匹配流程
```
1. 获取活动需求信息
2. 筛选符合条件的赞助
3. 计算各维度匹配分数
4. 加权计算总分
5. 按分数排序返回结果
```

### 8.2 Nginx配置示例
```nginx
server {
    listen 80;
    server_name linmeng.com;

    # 统一入口
    location / {
        root /var/www/portal;
        try_files $uri $uri/ /index.html;
    }

    # 社区端
    location /community {
        alias /var/www/community-h5;
        try_files $uri $uri/ /index.html;
    }

    # API代理
    location /api {
        proxy_pass http://localhost:3001;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```
