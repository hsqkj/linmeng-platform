## 四、Railway部署（后端+数据库）

### 4.1 创建Railway项目

1. 访问 https://railway.app
2. 点击 "Start a New Project"
3. 选择 "Deploy from GitHub repo"
4. 授权GitHub访问，选择 `linmeng-platform` 仓库
5. 选择 `server` 目录作为根目录

### 4.2 添加MySQL数据库

1. 在项目中点击 "+ New"
2. 选择 "Database" → "Add MySQL"
3. 数据库会自动创建

### 4.3 配置环境变量

在Railway项目设置中添加以下环境变量：

```
NODE_ENV=production
PORT=3001
JWT_SECRET=your_random_secret_key_here_at_least_32_characters
JWT_EXPIRES_IN=7d
```

**数据库变量（自动生成，需手动关联）：**

点击MySQL服务，在 "Variables" 标签页可以看到：
- `MYSQLHOST`
- `MYSQLPORT`
- `MYSQLUSER`
- `MYSQLPASSWORD`
- `MYSQLDATABASE`

在后端服务中添加这些变量的映射：
```
DB_HOST=${{MySQL.MYSQLHOST}}
DB_PORT=${{MySQL.MYSQLPORT}}
DB_USER=${{MySQL.MYSQLUSER}}
DB_PASSWORD=${{MySQL.MYSQLPASSWORD}}
DB_NAME=${{MySQL.MYSQLDATABASE}}
```

### 4.4 获取后端URL

部署成功后，Railway会提供一个域名，例如：
```
https://linmeng-api-production.up.railway.app
```

记录这个地址，前端配置需要使用。

## 五、Vercel部署（前端）

### 5.1 部署统一入口 (Portal)

1. 访问 https://vercel.com
2. 点击 "Add New..." → "Project"
3. 选择 `linmeng-platform` 仓库
4. 配置项目：
   - **Framework Preset**: Vue.js
   - **Root Directory**: `apps/portal`
   - **Build Command**: `npm run build`
   - **Output Directory**: `dist`
5. 添加环境变量：
   ```
   VITE_API_BASE_URL=https://你的railway后端地址
   ```
6. 点击 "Deploy"

## 九、常见问题

### Q1: Railway部署失败
检查日志，常见原因：
- 数据库连接失败：检查环境变量配置
- 依赖安装失败：检查package.json

### Q2: Vercel构建失败
- 检查Node.js版本
- 检查环境变量是否正确
- 查看构建日志

### Q3: API跨域问题
后端已配置CORS，如果仍有问题，检查：
- API地址是否正确
- 是否使用了HTTPS

### Q4: 数据库迁移失败
在Railway中手动执行：
```bash
npx sequelize-cli db:migrate
```
