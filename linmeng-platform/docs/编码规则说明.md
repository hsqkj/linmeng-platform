# 编码规则说明文档

## 编码格式设计原则
1. **可读性强**：编码包含前缀、日期、序号，便于人工识别
2. **唯一性保证**：通过日期+序号组合确保唯一
3. **可追溯性**：从编码可看出创建日期
4. **统一规范**：所有编码采用统一格式

## 编码规则详解

### 1. 社区编码 (community_code)
**格式**：`SQ` + `YYYYMMDD` + `NNNN`

**示例**：`SQ202603260001`

**说明**：
- `SQ`：社区前缀（She Qu）
- `YYYYMMDD`：创建日期（年月日）
- `NNNN`：当日序号（4位数字，从0001开始）

**生成时机**：社区用户注册成功时自动生成

### 6. 订单编号 (order_code)
**格式**：`DD` + `YYYYMMDDHHmmss` + `RRR`

**示例**：`DD20260326143052001`

**说明**：
- `DD`：订单前缀（Ding Dan）
- `YYYYMMDDHHmmss`：创建时间（年月日时分秒）
- `RRR`：随机数（3位数字，000-999）

**生成时机**：创建订单时自动生成

**特点**：
- 包含精确到秒的时间戳
- 加随机数避免冲突
- 适合高并发场景

## 技术实现

### 编码生成器位置
`server/src/utils/codeGenerator.js`

### 使用方法
```javascript
const CodeGenerator = require('../utils/codeGenerator')

// 生成社区编码
const communityCode = await CodeGenerator.generateCommunityCode()

// 生成商家编码
const merchantCode = await CodeGenerator.generateMerchantCode()

// 生成需求编码
const demandCode = await CodeGenerator.generateDemandCode()

// 生成赞助编码
const sponsorCode = await CodeGenerator.generateSponsorCode()

// 生成合作编码
const cooperationCode = await CodeGenerator.generateCooperationCode()

// 生成订单编号
const orderCode = await CodeGenerator.generateOrderNo()
```

### 数据库字段
所有编码字段均为 `VARCHAR(20)` 类型，存储在对应的表中：
- `community_user.community_code`
- `merchant_user.merchant_code`
- `activity.demand_code`
- `sponsor_infos.sponsor_code`
- `cooperation.cooperation_code`
- `order.order_code`

## 编码优势

1. **易于识别**：通过前缀即可知道编码类型
2. **便于查询**：可通过日期范围快速筛选
3. **防止重复**：日期+序号组合确保唯一性
4. **便于统计**：可按日期统计各类数据
5. **友好展示**：编码格式规范，便于用户理解

---

## 注意事项

1. 编码生成后不可修改
2. 编码生成失败时会返回默认编码（序号0001）
3. 每日序号从0001开始重新计数
4. 订单编号使用时间戳+随机数，适合高并发场景
5. 所有编码均为大写字母和数字组合
