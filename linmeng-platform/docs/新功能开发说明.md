# 新功能开发说明

## 一、社区端赞助留言功能

### 1. 数据库表结构
- **sponsor_comments表**：存储赞助留言和回复
  - comment_code: 留言编码
  - sponsor_id: 赞助信息ID
  - community_id: 社区ID
  - merchant_id: 商家ID
  - content: 留言内容
  - parent_id: 父留言ID（用于回复）
  - reply_to_id: 回复的留言ID
  - is_read: 是否已读

### 2. API接口
- POST /api/sponsor-comment/create - 创建留言
- GET /api/sponsor-comment/list/:sponsor_id - 获取留言列表
- GET /api/sponsor-comment/merchant/unread - 获取商家未读留言数
- GET /api/sponsor-comment/merchant/list - 获取商家留言列表
- PUT /api/sponsor-comment/:id/read - 标记为已读
- POST /api/sponsor-comment/reply - 商家回复留言

### 3. 前端功能
- 社区端：在赞助详情页添加留言输入框和留言列表
- 商家端：在消息中心显示留言提醒，可查看和回复留言

## 二、业务员渠道码功能

### 1. 数据库表结构
- **salesmen表**：业务员信息
  - salesman_code: 业务员编码
  - name: 姓名
  - phone: 手机号
  - password: 密码
  - channel_code: 渠道码
  - channel_link: 渠道链接
  - commission_rate: 提成比例
  - total_commission: 累计提成金额
  - status: 状态

- **commissions表**：提成记录
  - commission_code: 提成编码
  - salesman_id: 业务员ID
  - merchant_id: 商家ID
  - order_id: 订单ID
  - order_amount: 订单金额
  - commission_rate: 提成比例
  - commission_amount: 提成金额
  - status: 状态（待结算/已结算/已取消）

- **merchant_users表新增字段**：
  - salesman_id: 业务员ID

### 2. API接口
- POST /api/salesman/register - 业务员注册
- POST /api/salesman/login - 业务员登录
- GET /api/salesman/profile - 获取业务员信息
- GET /api/salesman/merchants - 获取渠道商户列表
- GET /api/salesman/commissions - 获取提成记录
- GET /api/salesman/stats - 获取统计数据
- GET /api/salesman/list - 后台获取业务员列表
- PUT /api/salesman/:id/status - 更新业务员状态
- PUT /api/salesman/:id/commission-rate - 更新提成比例
- GET /api/salesman/channel/:channel_code - 验证渠道码

### 3. 前端功能
- 业务员端：注册、登录、查看渠道码、查看商户、查看提成
- 商家端：注册时支持渠道码输入
- 管理后台：业务员管理、统计报表

## 三、功能流程

### 1. 留言流程
1. 社区查看赞助详情 → 点击留言 → 输入内容 → 提交
2. 商家收到留言提醒 → 查看留言 → 回复留言
3. 社区收到回复提醒 → 查看回复 → 继续对话

### 2. 渠道码流程
1. 业务员注册 → 生成专属渠道码和链接
2. 业务员分享渠道码/链接给商家
3. 商家通过渠道码注册 → 自动绑定业务员
4. 商家缴费 → 生成提成记录
5. 业务员查看提成统计
