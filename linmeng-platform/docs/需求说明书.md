## 一、项目概述

### 1.1 项目背景
社区活动常常面临资源匮乏的问题：缺少资金赞助、物资支持、场地资源、专业人才等。同时，许多商家愿意通过赞助社区活动来提升品牌形象、触达潜在客户，但缺乏有效的渠道找到合适的社区活动。

### 1.2 项目目标
打造一个智能匹配平台，连接社区与商家，实现：
- 社区活动获得所需资源支持
- 商家精准触达目标用户群体
- 招商大使通过推广获得收益
- 平台构建可持续的商业生态

### 1.3 目标用户
| 用户角色 | 描述 | 核心需求 |
|---------|------|---------|
| 社区用户 | 社区居委会、社区工作者 | 发布活动需求，获取商家赞助 |
| 商家用户 | 周边商户、企业 | 发布赞助信息，参与社区活动 |
| 招商大使 | 推广人员 | 推广商家入驻，获取提成 |
| 管理员 | 平台运营人员 | 审核管理、数据分析 |

## 四、数据需求

### 4.1 核心数据实体
| 实体 | 描述 | 主要字段 |
|-----|------|---------|
| 社区用户 | 社区端用户 | community_code, name, phone, area |
| 商家用户 | 商家端用户 | merchant_code, name, phone, business_name |
| 招商大使 | 推广人员 | salesman_code, name, phone, channel_code |
| 活动 | 社区活动 | demand_code, title, type, status |
| 赞助 | 商家赞助 | sponsor_code, title, type, status |
| 合作 | 合作记录 | cooperation_code, activity_id, sponsor_id |
| 评价 | 商家评价 | 评分、评价内容 |
| 消息 | 系统消息 | 类型、内容、状态 |
| 提成 | 提成记录 | 金额、状态、时间 |

### 4.2 数据关系
```
社区用户 1:N 活动
商家用户 1:N 赞助
活动 N:M 赞助 (通过合作记录)
招商大使 1:N 商家用户
商家用户 1:N 提成记录
```

## 六、约束条件

### 6.1 技术约束
- 前端：Vue 3 + Vite
- 后端：Node.js + Express
- 数据库：MySQL
- ORM：Sequelize

### 6.2 业务约束
- 社区用户需审核通过后才能发布活动
- 商家用户需审核通过后才能发布赞助
- 提成在商家付费后自动计算

### 6.3 法规约束
- 遵守《网络安全法》
- 遵守《个人信息保护法》
- 用户协议和隐私政策
