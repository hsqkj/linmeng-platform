# 商圈快讯功能说明书

## 1. 系统概述

### 1.1 系统名称
商圈快讯（Business District News）

### 1.2 系统定位
商圈快讯是一个面向多社区商圈的轻量化消息发布与互动平台，旨在帮助商家快速发布促销信息、活动通知等内容，并提供数据统计功能，降低商家的营销推广门槛，提升居民对商家的信任度，同时为居民提供便捷的商圈信息浏览、互动与消费体验。

### 1.3 主要功能
- 商家注册与认证
- 模板化消息发布
- 消息管理与查看
- 浏览统计与数据分析
- 居民端消息浏览与互动

### 1.4 技术架构
- **前端技术**：HTML5 + CSS3 + JavaScript
- **第三方库**：Lucide图标库、Chart.js图表库
- **数据存储**：浏览器本地存储（LocalStorage）
- **响应式设计**：支持移动端和桌面端

## 2. 功能模块详细说明

### 2.1 商家注册模块

#### 2.1.1 功能描述
允许商家用户注册账号，填写基本信息，用于后续的消息发布和管理。

#### 2.1.2 界面元素
- 商家名称输入框
- 负责人姓名输入框
- 手机号输入框与验证码
- 门店地址（系统将自动获取LBS位置，帮助您精准触达周边3公里客群）
- 商家类型选择（下拉菜单）
- 营业执照（该项用虚线框 注明可选）（上传营业执照，支持 PNG, JPG, WEBP 格式，大小不超过5MB。认证后可提升门店可信度）
- 提交按钮（注册并开始发布推广信息）
- 底部提示（注册即表示您同意《用户服务协议》《隐私政策》）（已有账号？立即登录）

#### 2.1.3 数据验证
- 所有字段为必填项

- 联系电话格式验证

  #### 2.1.4 数据处理

- 注册信息通过App.saveUserData()方法保存到本地存储
- 注册成功后自动登录并跳转至发布消息页面

### 2.2 消息发布模块

#### 2.2.1 功能描述
提供3步式模板化消息发布流程，帮助商家快速创建和发布商圈消息。

#### 2.2.2 操作流程
1. **选择模板**：从预设模板中选择消息类型（促销活动、通知公告、新品上市、店铺动态）
2. **编辑内容**：根据模板填写消息标题、内容、图片等信息
3. **发布设置**：选择消息有效期和发布范围

#### 2.2.3 界面元素
- 步骤指示器
- 模板选择区域（包含模板卡片）
- 消息编辑表单
- 有效期选择器
- 发布范围选择器
- 上一步/下一步/发布按钮

#### 2.2.4 数据处理
- 发布的消息通过App.saveMessage()方法保存到本地存储
- 自动生成消息ID、创建时间等系统字段
- 消息初始浏览量、点赞数、评论数等设为0

#### 2.2.5 发布后操作
- 显示发布成功提示
- 提供查看已发布消息和继续发布新消息的导航链接

### 2.3 消息查看模块

#### 2.3.1 功能描述
商家可以查看自己发布的所有消息，包括消息详情和互动数据。

#### 2.3.2 界面元素
- 消息列表
- 消息卡片（包含标题、内容摘要、创建时间、互动数据）
- 发布新消息按钮
- 查看统计按钮
- 评论展开/收起按钮
- 评论输入框

#### 2.3.3 功能特性
- 按发布时间倒序显示消息
- 每条消息显示浏览量、点赞数、评论数
- 支持展开查看完整评论列表
- 支持添加新评论

#### 2.3.4 数据处理
- 通过App.getUserMessages()获取当前用户发布的所有消息
- 评论通过App.addComment()方法保存到对应消息中
- 实时更新消息的互动数据

### 2.4 统计分析模块

#### 2.4.1 功能描述
为商家提供消息浏览和互动的统计数据，通过图表直观展示。

#### 2.4.2 统计指标
- 总浏览量
- 总点赞数
- 总评论数
- 总分享数

#### 2.4.3 图表类型
- 每日浏览量趋势图（折线图）
- 性别分布饼图
- 消息类别占比饼图
- 小时浏览量柱状图

#### 2.4.4 数据处理
- 通过App.getStatistics()获取统计数据
- 使用Chart.js库渲染各类图表
- 统计数据通过App.calculateStatistics()方法定期更新

### 2.5 居民端浏览模块

#### 2.5.1 功能描述
居民用户可以浏览商圈内所有商家发布的消息，并进行互动。

#### 2.5.2 界面元素
- 搜索框
- 分类筛选标签（全部、促销活动、通知公告、新品上市、店铺动态）
- 消息列表（居民端消息卡片）
- 互动按钮（点赞、评论、分享）
- 评论输入框
- 切换到商家端按钮

#### 2.5.3 功能特性
- 支持关键词搜索
- 支持按消息类别筛选
- 消息卡片显示商家信息、消息内容和互动数据
- 支持点赞、评论、分享操作
- 实时更新互动数据

#### 2.5.4 数据处理
- 显示所有商家发布的消息（按发布时间倒序）
- 互动操作通过App.updateMessageStats()方法更新数据
- 评论通过App.addComment()方法保存

## 3. 技术实现说明

### 3.1 核心数据结构

#### 3.1.1 用户数据结构
```javascript
{
  id: string,              // 用户唯一标识
  businessName: string,    // 商家名称
  contactName: string,     // 联系人姓名
  phone: string,           // 联系电话
  email: string,           // 电子邮箱
  password: string,        // 密码
  businessType: string     // 商家类型
}
```

#### 3.1.2 消息数据结构
```javascript
{
  id: string,              // 消息唯一标识
  title: string,           // 消息标题
  content: string,         // 消息内容
  category: string,        // 消息类别
  templateType: string,    // 模板类型
  image: string,           // 消息图片
  startTime: string,       // 开始时间
  endTime: string,         // 结束时间
  merchantId: string,      // 发布商家ID
  merchantName: string,    // 发布商家名称
  createTime: string,      // 创建时间
  views: number,           // 浏览量
  likes: number,           // 点赞数
  comments: Array,         // 评论列表
  shares: number           // 分享数
}
```

#### 3.1.3 统计数据结构
```javascript
{
  totalViews: number,      // 总浏览量
  totalLikes: number,      // 总点赞数
  totalComments: number,   // 总评论数
  totalShares: number,     // 总分享数
  dailyViews: Array,       // 每日浏览量
  genderDistribution: Object,  // 性别分布
  hourlyTraffic: Array,    // 小时流量
  categoryRatio: Object    // 类别占比
}
```

### 3.2 核心方法说明

#### 3.2.1 用户数据管理
- `App.saveUserData(userData)`: 保存用户数据到本地存储
- `App.loadUserData()`: 从本地存储加载用户数据
- `App.isLoggedIn()`: 检查用户是否已登录
- `App.logout()`: 用户登出，清除登录状态

#### 3.2.2 消息数据管理
- `App.saveMessage(message)`: 保存消息到本地存储
- `App.loadMessages()`: 从本地存储加载消息
- `App.getUserMessages()`: 获取当前用户发布的消息
- `App.updateMessageStats(messageId, type)`: 更新消息统计数据
- `App.addComment(messageId, comment)`: 添加评论到指定消息

#### 3.2.3 统计数据管理
- `App.calculateStatistics()`: 计算统计数据
- `App.updateStatistics(newMessage)`: 更新统计数据
- `App.getStatistics()`: 获取统计数据

### 3.3 本地存储键值
- `businessDistrictUser`: 存储用户数据
- `businessDistrictMessages`: 存储所有消息数据
- `businessDistrictStatistics`: 存储统计数据

## 4. 使用流程指南

### 4.1 商家注册与发布流程

1. **访问注册页面**：打开register.html，填写商家信息
2. **提交注册**：点击提交按钮完成注册
3. **进入发布页面**：注册成功后自动跳转至post-message.html
4. **选择模板**：从模板列表中选择合适的消息模板
5. **填写内容**：根据模板填写消息标题、内容等信息
6. **设置参数**：选择消息有效期和发布范围
7. **发布消息**：点击发布按钮完成消息发布
8. **查看消息**：发布成功后可以在view-messages.html查看已发布的消息
9. **查看统计**：在statistics.html查看消息的浏览和互动统计

### 4.2 居民浏览与互动流程

1. **访问居民端页面**：打开resident-view.html
2. **浏览消息**：查看商圈内所有商家发布的消息
3. **搜索筛选**：使用搜索框和分类标签筛选感兴趣的消息
4. **消息互动**：对感兴趣的消息进行点赞、评论或分享操作
5. **切换身份**：点击"切换到商家端"按钮可以切换到商家操作界面

## 5. 界面设计规范

### 5.1 色彩方案
- 主色调：紫色 (#4f46e5)
- 辅助色：绿色 (#10b981)、橙色 (#f59e0b)、红色 (#ef4444)、粉色 (#ec4899)
- 中性色：深灰 (#1f2937)、中灰 (#4b5563)、浅灰 (#6b7280)、超浅灰 (#f3f4f6)

### 5.2 排版规范
- 标题：使用较大字号和粗体，颜色为深灰色
- 正文：使用适中字号，颜色为中灰色
- 辅助文字：使用较小字号，颜色为浅灰色

### 5.3 布局规范
- 采用卡片式布局，提高内容可读性
- 响应式设计，适配不同屏幕尺寸
- 页面间距统一，保持视觉一致性

### 5.4 交互规范
- 按钮和可点击元素有明显的悬停效果
- 表单提交有明确的成功/失败反馈
- 页面切换有平滑的过渡效果

## 6. 浏览器兼容性

- Chrome 80+
- Firefox 75+
- Safari 13+
- Edge 80+

## 7. 注意事项

1. 系统使用浏览器本地存储，清除浏览器数据会导致所有数据丢失
2. 建议使用最新版本的现代浏览器以获得最佳体验
3. 统计数据部分使用模拟数据，实际应用中需要连接后端API获取真实数据
4. 居民端的分享功能为模拟实现，实际应用中需要集成第三方分享SDK
